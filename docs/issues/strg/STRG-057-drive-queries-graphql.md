# STRG-057: Implement Drive GraphQL queries

## Summary

Implement `drives` (paginated connection) and `drive(id)` queries under the `StorageQueries` namespace. Returns drives accessible to the authenticated user within their tenant. `Drive.providerConfig` and `Drive.tenantId` are never exposed.

## Technical Specification

### Schema (under `query { storage { ... } }`):

```graphql
type StorageQueries {
  drives(first: Int, after: String, last: Int, before: String): DriveConnection!
  drive(id: ID!): Drive
}

# Relay cursor connection (generated by HC)
type DriveConnection {
  nodes: [Drive!]!
  pageInfo: PageInfo!
  totalCount: Int!
}
```

`drives` returns a `DriveConnection` with Relay cursor pagination ŌĆö not a plain `[Drive!]!` list.

### File: `src/Strg.GraphQL/Queries/DriveQueries.cs`

```csharp
[ExtendObjectType<StorageQueries>]
public sealed class DriveQueries
{
    [UsePaging(DefaultPageSize = 50, MaxPageSize = 200)]
    [GraphQLComplexity(5)]
    [Authorize(Policy = "FilesRead")]
    public IQueryable<Drive> GetDrives(
        [Service] StrgDbContext db,
        [GlobalState("tenantId")] Guid tenantId)
        => db.Drives.Where(d => d.TenantId == tenantId)
                    .OrderBy(d => d.Name);

    [Authorize(Policy = "FilesRead")]
    public Task<Drive?> GetDrive(
        ID id,
        [Service] StrgDbContext db,
        [GlobalState("tenantId")] Guid tenantId,
        CancellationToken ct)
        => db.Drives.FirstOrDefaultAsync(
               d => d.Id == (Guid)id && d.TenantId == tenantId, ct);
}
```

### Usage examples:

```graphql
# List all drives with pagination
query {
  storage {
    drives(first: 20) {
      nodes { id name providerType isDefault isEncrypted }
      pageInfo { hasNextPage endCursor }
      totalCount
    }
  }
}

# Single drive with nested file listing
query {
  storage {
    drive(id: "...") {
      id name
      files(first: 10, filter: { isFolder: false }) {
        nodes { id name size mimeType }
        pageInfo { hasNextPage }
      }
    }
  }
}
```
